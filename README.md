## 一、实验目的
本实验旨在通过实践操作,掌握基于云计算环境的网站部署技能。具体包括:构建环境并实现个人博客/网站的搭建,撰写实验报告,并将这份报告作为一篇博文发布在自己的博客 / 网站上。

## 二、实验环境
### 硬件环境
联网的计算机一台
### 软件环境
Windows or Linux

## 三、实验步骤与内容
本次实验采用GitHub Pages 免费静态页面托管服务实现个人博客搭建与实验报告发布，具体步骤如下：
1. **GitHub 账号注册**
打开 GitHub 官网(https://github.com),点击「Sign up」,输入邮箱、自定义用户名和密码,完成邮箱验证后即可注册成功,全程无收费项、无需信用卡绑定,学生可直接使用。
2. **创建 GitHub Pages 专属仓库**
登录账号后点击右上角「+」→「New repository」,进入仓库创建页面,核心配置:仓库名必须为**用户名.github.io**(用户名与注册账号一致,后缀固定),可见性选择「Public」(免费版仅支持公共仓库),其余默认,点击「Create repository」完成创建,该仓库作为博客的云端存储空间。
3. **搭建个人博客基础页面**
进入新建仓库,点击顶部「Settings」,在左侧菜单栏找到「Pages」,在「Build and deployment」区域设置:来源选「Deploy from a branch」,分支选默认主分支(main/master),文件夹选「/root」,点击「Save」;配置Jekyll主题为minimal,GitHub 自动完成博客部署,等待 1-2 分钟,页面显示绿色提示「Your site is published at...」,即博客搭建成功,访问该链接可进入博客首页。
4. **撰写实验报告并发布为博文**
回到仓库首页,点击「README.md」文件右侧的编辑铅笔图标,进入编辑页面;将实验报告完整内容按 Markdown 格式编辑(标题用#/##、分段用换行、列表用-),编辑完成后拉至页面底部,填写提交说明,点击「Commit changes」完成提交;刷新博客访问地址,实验报告已自动发布为博客首页核心内容。
5. **验证博客访问效果**
在浏览器中多次访问博客公网地址,确认页面正常加载、实验报告内容完整无错乱;可在不同设备(手机、平板)测试访问,验证GitHub Pages的跨设备、跨网络兼容性。

## 四、结论分析与体会
1. GitHub Pages提供了优质的云计算实践载体,其零成本、低门槛的特点,让我们无需关注底层服务器配置、网络部署等复杂问题,能专注于网站部署核心技能的学习,完美适配高校云计算课程的实验需求;
2. 云计算实操中细节把控至关重要,实验中发现,仓库名命名不规范、Pages分支设置错误、Markdown格式混乱、Jekyll主题配置错误等细节问题,都会导致博客无法访问或内容排版错乱,这让我认识到,即使是简易的云服务操作,也需要严谨的操作习惯,每一步完成后及时验证效果;
3. 本次实验让我对云计算的应用价值有了更直观的理解,云计算将海量的服务器资源封装为简单易用的云端服务,让普通用户无需具备专业的运维知识,就能快速获取计算资源、实现网站部署,真正体现了云计算「按需分配、普惠化」的优势;
4. 实验实现了「实操-总结-发布」的闭环,从搭建博客的实践操作,到整理实验过程撰写报告,再到将报告发布为博文,不仅提升了云计算实操能力,也锻炼了文档整理和内容发布的综合能力,为后续云计算相关技术的学习打下了实践基础。
